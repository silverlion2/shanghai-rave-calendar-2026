# Shanghai Rave Index 自媒体获客计划

制定日期：2026-06-08  
适用项目：Shanghai Rave Index  
周期：90 天试运营，之后按数据复盘重排渠道预算

## 1. 目标与基本判断

Shanghai Rave Index 的核心增长机会不是做泛夜生活账号，而是成为「上海电子音乐周末决策入口」：用户想知道今晚、这周末、下个月去哪里，平台给出可信来源、DJ/场地背景、日历保存和活动深链。

90 天内验证三个假设：

1. 小红书和抖音能带来新的本地兴趣用户，尤其是搜索「上海周末去哪」「上海 techno」「上海 club」「上海 rave」的人群。
2. 微信公众号、微信群、视频号能沉淀复访用户，形成每周五固定打开的习惯。
3. 场地、主办方、DJ 愿意转发被收录的活动页面，因为这能给他们带来更清晰的活动介绍和可分享链接。

北极星指标：每周有效到站用户数，以及用户是否完成至少一个强意图动作：打开活动深链、点击详情/购票、导出 `.ics`、保存/分享内容、加入私域更新渠道。

初始 90 天目标，前两周后按真实基线修正：

- 第 30 天：累计 1,000 次网站访问，至少 150 次活动深链访问。
- 第 60 天：累计 3,000 次网站访问，建立 1 个可持续更新的微信私域入口。
- 第 90 天：累计 6,000 次网站访问，完成 8-12 次场地、主办方或 DJ 转发合作。

## 2. 用户分层

核心用户：

- 上海本地电子音乐听众，20-35 岁，关注 techno、house、bass、industrial、trance、warehouse party。
- 在沪外籍/双语人群，常用 Instagram、小红书、微信群和 Resident Advisor。
- 来上海短住或旅游的人，搜索「this weekend in Shanghai」「Shanghai techno」「上海夜生活」。

供给侧用户：

- 场地、主办方、厂牌、摄影师、DJ、视觉艺术家。
- 他们的目标是卖票、建立场景口碑、让阵容和 set time 被更准确传播。

## 3. 定位与传播句

中文定位：

> 上海电子音乐周末日历：可溯源的 rave、techno、warehouse、bass 活动清单。

英文定位：

> A sourced Shanghai electronic music calendar for techno, rave, warehouse, bass, and underground events.

每条内容都尽量落在一个清晰承诺上：

- 今晚去哪：少废话，给 3 个选择。
- 这周末去哪：按 vibe、场地、时间和票务状态整理。
- 为什么值得去：介绍 DJ、厂牌、场地、声音方向。
- 怎么去和怎么保存：给活动链接、`.ics`、地图/交通提示。

## 4. 渠道优先级

### A. 小红书：主阵地，用来拿搜索和收藏

目标：吃下「城市周末」「夜生活」「小众音乐」「新上海人」搜索流量。

内容形式：

- 图文笔记：每周 4-6 条。
- 短视频：每周 2-3 条，可复用抖音和视频号素材。
- 关键词布局：上海周末去哪、上海夜生活、上海 techno、上海 rave、上海 club、上海 live、上海电子音乐、上海蹦迪、warehouse party。

固定栏目：

- `这周末上海电子音乐去哪`
- `今晚 3 个选择`
- `上海场地速查`
- `DJ / 厂牌 60 秒介绍`
- `第一次去上海 underground party 要知道什么`

转化动作：

- 主页链接放 Vercel 主站或轻量 link-in-bio。
- 评论区引导「看主页日历」或「私信关键词：周末」。
- 每条笔记带一个具体页面，而不是只放首页，例如 DJ 数据库、场地页、具体活动深链。

### B. 抖音：短视频触达，用来测爆点

目标：用高频短内容触达本地娱乐兴趣人群，再把有效内容导向日历。

内容形式：

- 每周 3-5 条 15-35 秒短视频。
- 开头 2 秒必须是选择题或时间压力，例如「这周六上海 techno 有 3 场」「今晚不想查票的话，看这个」。
- 画面优先用站内 poster、活动卡片、场地外景、公开授权素材，不使用未经授权的人脸近景。

投放原则：

- 先自然发布 2 周，找出收藏率、完播率、主页点击最好的前 20% 内容。
- 只加热已经证明有效的内容。小预算测试即可，目标不是买粉，而是验证「内容主题 -> 到站」链路。
- 如果做付费，优先用本地兴趣、同城、音乐/演出/夜生活相关人群。

### C. 微信：沉淀复访，用来做每周固定入口

目标：把一次性浏览变成每周复访。

最小配置：

- 公众号：每周五 18:00 发「本周末上海电子音乐日历」。
- 视频号：同步抖音短视频中表现最好的内容。
- 微信群或企业微信：用作周更提醒、活动纠错、主办方投稿。

公众号内容结构：

1. 本周推荐 3-5 场。
2. 按 vibe 分组：techno / house / bass / experimental / trance。
3. 每场只写关键理由、时间、地点、票务/详情链接。
4. 底部固定放「提交活动」「查看完整日历」「DJ 数据库」。

私域规则：

- 不刷屏。每周固定 1-2 次更新，临时变动只发重要票务/时间更正。
- 群内禁止未经确认的活动传闻，和网站 source rules 保持一致。
- 鼓励主办方自荐，但必须提供可验证来源。

### D. Instagram / X：双语补充，用来覆盖外籍和国际 DJ 圈

目标：让英文受众知道这是一个更新中的 Shanghai calendar。

内容形式：

- Instagram：每周 3 条 post/reel，4-8 条 story，重点做活动倒计时和 repost。
- X：每周 3-5 条短帖，适合发布「newly added」「this weekend」「DJ profile」。
- 每个英文帖都带一个明确链接或截图二维码，减少用户搜索成本。

### E. B 站：低频沉淀，用来做场景文化内容

目标：不是短期获客主力，而是做可搜索的长尾内容。

内容形式：

- 每月 1-2 条 3-8 分钟视频。
- 主题如「上海 techno 场地地图」「第一次去 underground party 怎么选」「一个 DJ set time 怎么看」。

## 5. 内容支柱

1. 周末决策内容：每周四/五发布，直接回答「去哪」。
2. 场地与主办方档案：把已有 `venues.html` 内容拆成小红书图文和短视频。
3. DJ 数据库拆条：把 `djs.html` 中的阵容、风格、关联活动做成短内容。
4. 新增活动提醒：对应网站 `Newly Added` 视角。
5. 场景教育：票务、set time、dress code、交通、寄存、入场、摄影礼仪。
6. 来源透明：解释 confirmed / watch-only，建立可信度。
7. UGC 与合作转发：只使用获得授权的现场图、海报、视频和 repost。

## 6. 每周执行日历

周一：

- 更新活动数据和本周候选清单。
- 小红书发「本周新增活动」。
- X/Instagram 发 newly added thread/story。

周二：

- 发 1 条场地或主办方介绍。
- 联系 5 个本周相关主办方、DJ 或场地请求互相转发。

周三：

- 发 1 条 DJ / 音乐风格科普。
- 剪 2 条短视频，准备周四/周五发布。

周四：

- 小红书、抖音、视频号发「这周末去哪」。
- Instagram story 做倒计时和活动链接。

周五：

- 公众号 18:00 发周末日历。
- 微信群同步精简版。
- 抖音/小红书发「今晚 3 个选择」。

周六：

- 发 last-minute 票务或 set time 更新。
- 只推确认信息，不扩散未证实消息。

周日：

- 发下周预告或场景复盘。
- 整理本周数据：曝光、收藏、评论、链接点击、到站、`.ics` 导出。

## 7. 90 天路线图

### 第 1 周：搭建追踪与模板

- 所有平台统一账号名、头像、简介和链接。
- 建立 UTM 规则：`utm_source=xiaohongshu|douyin|wechat|instagram|x`，`utm_campaign=weekend_digest_YYYYMMDD`。
- 建 5 套 Canva/Figma 模板：周末清单、今晚推荐、DJ 卡、场地卡、新增活动。
- 网站增加或准备一个固定入口：`Join weekly updates`，先用微信二维码或表单承接。
- 建立投稿表单：主办方提交活动名、时间、场地、票务链接、官方海报、source link。

### 第 2-4 周：自然流量验证

- 小红书至少 18 条笔记。
- 抖音至少 12 条短视频。
- 公众号至少 3 篇周更。
- 每周联系 20 个供给侧账号。
- 记录每条内容的 hook、形式、发布时间、关键词、转化数据。

判断标准：

- 小红书收藏率和评论求链接数量。
- 抖音完播率、主页访问、私信数量。
- 微信周更阅读完成率和群内反馈。
- 网站活动深链和 `.ics` 导出是否随发布峰值上涨。

### 第 5-8 周：放大有效内容

- 将前 4 周表现最好的 5 个主题做系列化。
- 对表现最好的小红书/抖音内容做小额加热测试。
- 每周做 1 次「合作卡片」：和 DJ、场地或厂牌共创。
- 给主办方提供可转发素材：海报截图 + 日历链接 + 一句话介绍。

建议预算：

- 低预算：0-1,000 RMB/月，全部靠自然内容和合作转发。
- 测试预算：3,000-5,000 RMB/月，60% 加热优质内容，20% 微型 KOC/摄影合作，20% 设计/工具/小福利。
- 不建议在没有内容胜率前直接大额投放。

### 第 9-12 周：形成供给侧飞轮

- 发布「每月上海电子音乐地图」长内容。
- 建立主办方月度投稿提醒。
- 做 2-3 次 DJ takeover 或 selector picks。
- 设计基础商业合作包：赞助周更、活动置顶、场地专题、DJ 数据库展示，但保持 source transparency，不卖虚假推荐。

## 8. 平台内容模板

小红书图文标题：

- `这周末上海 techno / rave 去哪：X 场按 vibe 分好`
- `第一次去上海 underground party，先看这 7 件事`
- `上海电子音乐场地速查：Abyss / Heim / System / Wigwam 怎么选`
- `今晚不想翻票务平台：3 个上海电子音乐选择`

抖音/视频号脚本：

```text
0-2s：这周六上海 techno 有 3 个选择。
3-8s：第一场：场地 + vibe + 适合谁。
9-15s：第二场：阵容/主办方 + 亮点。
16-24s：第三场：更 late / 更 heavy / 更 chill 的选择。
25-30s：完整日历在主页，能直接保存到手机日历。
```

公众号开头：

```text
这周末的上海电子音乐清单已更新。
下面按 vibe 分组，只放已确认来源或有明确票务/官方页面的活动。完整日历和 DJ 数据库在文末。
```

合作私信：

```text
你好，我们在维护 Shanghai Rave Index，一个可溯源的上海电子音乐活动索引。
这周已收录你们的活动：[event link]
如果时间、票务、阵容或 set time 有更新，可以直接发官方链接给我们。我们也可以给你们一张双语活动卡，方便转发。
```

## 9. 转化与产品建议

当前站点已经有活动深链、场地页、DJ 数据库和 `.ics` 导出，适合承接自媒体流量。为了让获客闭环更完整，建议新增或补齐：

- 首页显眼位置放「每周更新入口」：微信二维码、公众号、群或表单。
- 每个活动页增加「分享文案复制」和「投稿纠错」入口。
- 给场地页和 DJ 页加可复制短链接，方便小红书/微信使用。
- 为小红书、抖音、微信、Instagram/X 使用不同 UTM 链接。
- 埋点统计：页面访问、活动详情点击、购票/详情外链点击、`.ics` 导出、分享按钮点击。

## 10. KPI 看板

每周记录：

- 内容产出：各平台发布数量。
- 触达：播放量、阅读量、曝光量。
- 互动：点赞、收藏、评论、转发、私信。
- 意图：主页访问、链接点击、评论求链接、私信关键词。
- 到站：按 UTM 区分的网站访问和活动深链访问。
- 转化：详情/购票外链点击、`.ics` 导出、加入微信/公众号、投稿数量。
- 供给侧：新增合作账号、主办方转发、纠错/补充信息次数。

优先放大这样的内容：

- 小红书收藏率高，说明有决策价值。
- 抖音完播率高且主页点击高，说明 hook 和 CTA 有效。
- 微信转发率高，说明适合群传播。
- 能带来活动深链访问，而不是只带来泛曝光。

## 11. 风险边界

- 只把 X/Twitter、小红书、抖音、Instagram、微信群消息作为 discovery lead，活动入库仍需 RA、SmartShanghai、官方场地/主办方/票务等更强来源确认。
- 不传播未确认 set time、秘密地址、私人群二维码或仅限邀请的信息。
- 不使用未经授权的人脸近景、现场视频或摄影作品。
- 内容表达聚焦音乐、场地、文化和安全，不使用违法暗示、药物暗示或夸张刺激话术。
- 明确年龄和入场限制；不面向未成年人营销夜间活动。
- 商业合作必须标注，不把付费展示伪装成编辑推荐。
